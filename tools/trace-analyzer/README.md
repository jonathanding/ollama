# Ollama Trace Analyzer

A profiling and visualization toolkit for Ollama inference traces. Captures per-operator execution data during model inference and provides interactive tools to explore performance bottlenecks, compare configurations, and generate LLM-ready analysis reports.

## Pre-built Release (Windows x64)

A ready-to-use Windows x64 package is available under `release/`. It includes a tracing-enabled Ollama binary, all native libraries, and the Trace Analyzer with a pre-built web UI — no building required. See the `README.md` inside the release package for usage instructions.

## Why

When running LLM inference on different hardware (CUDA, Vulkan, Metal, CPU), understanding where time is spent is critical for optimization. Raw traces contain thousands of operator events — this tool turns them into actionable insights:

- **DAG View**: Interactive computation graph with layer folding, heatmap coloring, and execution replay
- **Timeline View**: Swimlane visualization of operator execution across passes
- **Compare View**: Side-by-side comparison of traces from different backends/configs
- **Hotspot Panel**: Ranked lists of the most expensive operators and data transfers
- **Markdown Reports**: LLM-ready text summaries you can paste into Claude for analysis

## Installation

Requires Python 3.10+.

```bash
pip install -e .
```

This installs the `ollama-trace-analyzer` CLI command with all dependencies (polars, click, jinja2).

## Collecting Traces

Set the `OLLAMA_TRACE_DIR` environment variable before starting Ollama:

```bash
# Linux/macOS
export OLLAMA_TRACE_DIR=/tmp/traces
ollama serve

# Windows (PowerShell)
$env:OLLAMA_TRACE_DIR = "C:\traces"
ollama serve
```

Then send an inference request as usual. Trace files (JSONL) will be written to the specified directory.

## Usage

### Generate a summary

```bash
ollama-trace-analyzer summary trace_req1.jsonl -o summary.json
```

### Compare two traces

```bash
ollama-trace-analyzer compare trace_cuda.jsonl trace_vulkan.jsonl \
    --labels "CUDA,Vulkan" -o compare.json
```

### Generate an LLM-ready report

```bash
# Single trace
ollama-trace-analyzer report trace_req1.jsonl -o report.md

# Comparison report
ollama-trace-analyzer report trace_cuda.jsonl \
    --compare trace_vulkan.jsonl --labels "CUDA,Vulkan" -o compare_report.md
```

### Export to Perfetto / chrome://tracing

```bash
ollama-trace-analyzer export trace_req1.jsonl -o trace.perfetto.json
# Open in https://ui.perfetto.dev/ or chrome://tracing
```

Each backend (CPU, CUDA, Vulkan) appears as a separate lane. Passes are shown as spanning events. Op args include tensor name, shape, dtype, and pass/sequence IDs.

### Launch the visualization server

```bash
ollama-trace-analyzer serve --data-dir /path/to/traces/
```

Opens a local server at `http://localhost:8765`. The browser UI provides interactive DAG, timeline, and comparison views. The `--data-dir` should contain `.json` files generated by the `summary` and `compare` commands.

Options:
- `--data-dir` (required): Directory containing summary/compare JSON files
- `--port` (default: 8765): Server port

## Typical Workflow

```bash
# 1. Collect a trace
OLLAMA_TRACE_DIR=/tmp/traces ollama serve &
curl http://localhost:11434/api/generate -d '{"model":"llama3:8b","prompt":"Hello"}'

# 2. Generate summary JSON
ollama-trace-analyzer summary /tmp/traces/trace_*.jsonl -o /tmp/traces/summary.json

# 3. Visualize in browser
ollama-trace-analyzer serve --data-dir /tmp/traces/

# 4. Export for Perfetto (interactive flame chart)
ollama-trace-analyzer export /tmp/traces/trace_*.jsonl -o trace.perfetto.json
# Open trace.perfetto.json in https://ui.perfetto.dev/

# 5. Or generate a report for LLM analysis
ollama-trace-analyzer report /tmp/traces/trace_*.jsonl -o report.md
# Paste report.md into Claude and ask: "Analyze this trace and identify bottlenecks"
```

## CLI Reference

| Command   | Description                         | Key Options                                |
|-----------|-------------------------------------|--------------------------------------------|
| `summary` | Single trace → summary.json         | `-o`, `--model`, `--pass`                  |
| `compare` | Two traces → compare.json           | `--labels` (required), `--threshold`, `-o` |
| `report`  | Trace → Markdown report             | `--compare`, `--labels`, `-o`              |
| `export`  | Trace → Perfetto/chrome://tracing   | `-o` (default: `<file>.perfetto.json`)     |
| `serve`   | Launch visualization server         | `--data-dir` (required), `--port`          |
